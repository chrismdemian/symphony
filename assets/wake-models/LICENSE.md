# Wake-word model licensing

## `hey-symphony.onnx`

**License:** Apache-2.0 (same as openWakeWord's training code)

This file is a model trained from scratch using:

- The Apache-2.0-licensed openWakeWord training stack and shared speech-embedding backbone (`dscripka/openWakeWord`)
- Synthetic positive samples generated by `piper-sample-generator` (Rhasspy, MIT) using Piper TTS voices that ship under permissive licenses
- Pre-embedded negative mel-spectrogram features from `davidscripka/openwakeword_features` (HuggingFace dataset, Apache-2.0)

**The CC BY-NC-SA 4.0 restriction on openWakeWord's bundled pretrained models (`hey_jarvis`, `alexa`, `hey_mycroft`, `hey_rhasspy`) does NOT apply to this artifact.** Those weights were trained by upstream on their own data; we trained ours from scratch.

## Reproducibility

To retrain from the same recipe:

1. Read `scripts/train-wake-word/README.md` for the end-to-end runbook
2. `scripts/train-wake-word/training-config.json` (saved alongside this file) carries the exact hyperparameters used for the current shipped model
3. Identical seed + identical voices + identical openWakeWord version = byte-identical output (within floating-point determinism)

## Future model swaps

If you upgrade to `livekit-wakeword` (community-validated 100× FA reduction, watch list per `research/voice-stack-research.md`) or train a different phrase:

1. Drop the new `.onnx` in this directory under a new name
2. Bump `assets/wake-models/CHECKSUMS.txt`
3. Update `voice.wakeWordModel` default in `src/utils/config-schema.ts`
4. The model's license should remain permissive (Apache-2.0 or MIT) so this LICENSE.md stays accurate
