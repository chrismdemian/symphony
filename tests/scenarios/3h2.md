# Phase 3H.2 production scenario

End-to-end exercise of the editable settings popup through `runStart`,
with a fake Maestro and fake RPC. The 3H.1 scenario covered read-only
display and source-line behavior; 3H.2 adds the editor state machine
+ disk persistence + cross-call serialization.

This scenario validates:

- Booting the launcher with no config on disk → popup opens with
  defaults → Enter on `modelMode` cycles `mixed → opus` → disk file
  is written with `modelMode: opus` → popup re-render shows `opus`
  with `(from file)` annotation.
- Two rapid Enter presses on the modelMode row toggle twice (audit C2
  rapid-fire correctness): `mixed → opus → mixed`. Disk reflects the
  final `mixed`.
- Esc closes the popup → chat placeholder restored.

Mirrors `tests/scenarios/3h1.test.ts`'s harness shape: same
FakeMaestroProcess, same makeFakeRpc, same makeTtyStream. Differs in
the keystrokes and disk-state assertions.

## Steps

1. Boot the launcher with `SYMPHONY_CONFIG_FILE` pointing at a
   non-existent tmp path.
2. Settle.
3. Send `/config Enter` to push the settings popup.
4. Settle.
5. Assert the popup is open with default `mixed` rendered.
6. Send `\r` (Enter) ONCE on the modelMode row (which is the default
   selection on mount).
7. Settle.
8. Assert the popup re-renders with `opus` AND the disk file now
   contains `modelMode: "opus"`.
9. Send a SECOND `\r` (Enter) — this should cycle back to mixed
   even though no settle happened between (testing the audit C2
   rapid-fire serialization in production conditions).
10. Settle.
11. Assert the popup shows `mixed` AND disk file `modelMode: "mixed"`.
12. Send `\x1b` (Esc) to close.
13. Settle. Assert chat placeholder restored.
14. Stop the launcher cleanly.

## Acceptance assertions

- After step 5: popup contains `Settings`, `Phase 3H.2`, `mixed`,
  section headers.
- After step 8: popup body shows `opus` and on-disk JSON has
  `modelMode: "opus"`.
- After step 11: popup body shows `mixed` (back to start) and on-disk
  JSON has `modelMode: "mixed"`. Two rapid presses cycled twice — not
  a no-op.
- After step 13: chat placeholder text rendered (popup unmounted).
- After teardown: `rpc.close` called.
