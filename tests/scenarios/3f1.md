# Phase 3F.1 production scenario

End-to-end exercise of the command palette + help overlay through
`runStart`, with a fake Maestro process and a fake RPC client. Validates
the full keystroke pipeline:

- StatusBar mounts → Ctrl+P pushes the palette popup → palette mounts
  with full command list → typed filter narrows the list with char-level
  highlight ANSI present → Enter invokes the focused command and pops
  the popup → main split returns.
- `?` (help shortcut) pushes the help overlay → grouped headers render
  → Esc pops back to the main split.
- Ctrl+W (worker selector) pushes the worker-select popup → typed
  filter narrows to a single worker → Enter selects via
  `WorkerSelectionProvider` → focus advances to the workers panel.

Mirrors `tests/scenarios/3e.test.ts`'s shape — same FakeMaestroProcess,
same `runStart` boot, same Ctrl-byte injection through the launcher's
stdin pipeline. Differs only in which RPC slices are mocked (workers
list non-empty so Ctrl+W is enabled).

## Steps

1. Boot the launcher with a fake Maestro and a fake RPC, two registered
   workers, no questions, no tasks.
2. Settle one polling tick.
3. Assert StatusBar contains `Workers: 2`.
4. Write `\x10` (Ctrl+P) to stdin.
5. Settle.
6. Assert stdout contains `Command palette` header AND a row showing
   `next panel` (the global Tab cycle).
7. Type `next` to filter.
8. Settle.
9. Assert stdout contains `next panel` AND violet ANSI escape sequence
   `\x1b[38;2;124;111;235m` (matched-char highlight).
10. Write `\x1b` (Esc) to dismiss palette.
11. Settle.
12. Assert palette no longer in current frame; chat panel back.
13. Write `?` (help-overlay shortcut). Help is now `'main'`-scoped
    so it fires only when chat/workers/output is the active scope —
    chat is active by default after step 12.
14. Settle.
15. Assert stdout contains `Help · keybinds` header AND scope group
    label `Global`.
16. Write `\x1b` (Esc) to close help.
17. Settle.
18. Write `\x17` (Ctrl+W) to open the worker selector.
19. Settle.
20. Assert stdout contains `Select worker` header + at least one
    `frontend redesign` (one of the registered worker intents).
21. Stop the launcher cleanly.

## Acceptance assertions

- After step 3: `StatusBar` cell `Workers: 2` rendered.
- After step 6: rendered stdout (most-recent slice) contains both the
  `Command palette` header and a recognized command title.
- After step 9: rendered stdout contains a violet truecolor ANSI run
  (palette match highlight).
- After step 12: rendered stdout contains chat panel marker (no popup).
- After step 15: rendered stdout contains `Help · keybinds`.
- After step 20: rendered stdout contains `Select worker`.
- `rpc.close()` called on launcher stop.
