# Phase 1C — Production Scenario

`WorktreeManager` creates a real git worktree from a real git repo, preserves a gitignored `.env` file into it, and a real `claude -p` worker (Phase 1B `WorkerManager`) spawned inside that worktree reads the preserved file and reports it via the structured completion contract.

## Given

- A `claude` binary is resolvable via PATH (`claude --version` returns 0). If not, this scenario **skips with a warning** so CI without claude still passes.
- A fresh temp directory containing an initialized git repo with one commit and a `.gitignore` that excludes `.env`.
- A `.env` file written to the project root after the initial commit (so it is gitignored, not tracked).
- The Phase 1C `WorktreeManager` and Phase 1B `WorkerManager` public APIs.

## When

1. Caller initializes the temp git repo (`git init`, one commit) and writes `.env` containing a known secret value to the project root.
2. Caller invokes `WorktreeManager.create({ projectPath, workerId, shortDescription })` to produce a managed worktree under `<projectPath>/.symphony/worktrees/<workerId>`.
3. Caller asserts that the preserve-copy step delivered `.env` into the worktree.
4. Caller spawns a `claude -p` worker inside that worktree with a prompt instructing the worker to (a) read `.env`, (b) emit a structured completion JSON fence whose `did[0]` echoes the secret value verbatim.
5. Caller drains the worker's event stream and awaits exit.
6. Caller invokes `WorktreeManager.removeIfClean(...)` to confirm the worktree comes out clean (worker only read; no file mutations).

## Then

Observable state:

- `info.path` exists on disk and equals `<projectPath>/.symphony/worktrees/<workerId>`.
- `info.branch` matches `symphony/<workerId>/<slug>`.
- `<info.path>/.env` exists and its content is byte-equal to the source `.env`.
- `<info.path>/.git/info/exclude` contains every default git-exclude pattern (`.agent_context`, `CLAUDE.md`, `AGENTS.md`, `.symphony/`, `.claude/`).
- Worker exits with `status === 'completed'` and `exitCode === 0`.
- Worker emits exactly one `structured_completion` event whose `report.did[0]` contains the known secret value (proves the worker read the preserved file).
- No `parse_error` events.
- `removeIfClean(info.path)` returns `true` and the worktree directory is gone.
- The branch the worktree was on no longer appears in `git branch --list`.

## Ground Truth Is Observable

Every assertion is derived from filesystem state Symphony observes (file exists / contents) or from event stream output the parser produced. The worker's textual reasoning is irrelevant — only the structured completion's `did` array is consulted, because that is the contract Phase 4E will lock for Maestro.

## Gates

- **Skip-gracefully**: If `claude --version` fails (non-zero exit or missing binary), the test is marked `skip` with the reason logged. `pnpm verify` still passes.
- **Timeout budget**: 90 seconds for the worker turn (cold-start friendly).
- **Network dependency**: Real claude spawns talk to Anthropic's API. Running this scenario locally consumes a small amount of Max-subscription quota — acceptable for a single acceptance check.
