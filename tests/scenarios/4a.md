# Phase 4A scenario — worker role templates reach the spawned worker

## Goal

End-to-end verification that when Maestro delegates work via the
`spawn_worker` MCP tool, the spawned worker's first message is the
**composed role prompt** (role opener + common suffix + `# Your Task`),
not the bare task description. Ground truth is the bytes the lifecycle
hands the child — never a worker self-report.

## Given

- A registered project at a temp git repo (real `git init`, one commit).
- The real frozen v1 artifacts at `research/prompts/` (no override).

## When / Then

### Track 1 — deterministic (always runs)

Drive the REAL `spawn_worker` tool handler (the exact entrypoint the MCP
dispatch calls) wired to a REAL `createWorkerLifecycle` + REAL
`WorkerRegistry`, with the lifecycle's OS boundary (the `claude`
subprocess) stubbed to capture `cfg.prompt`.

1. **researcher** — `spawn_worker({role:'researcher', task_description})`.
   The captured child prompt contains `## Your Role: Researcher`, the
   `read-only investigator` fence, the common suffix's
   `### Reporting Format — MANDATORY` + `"audit": "PASS"` contract, the
   substituted worktree path, and a trailing
   `# Your Task\n\n<task_description>` block. No leaked `{tokens}` or
   `## BEGIN SUFFIX` meta fence.
2. **implementer** — same call with `role:'implementer'`; prompt carries
   the implementer opener (`execute an approved plan`) and the same
   suffix + task block.
3. **scope-clamp invariant** — the composed prompt contains the verbatim
   scope-clamp sentence (regression guard for rule #7 propagation).

### Track 2 — real `claude -p` (skips if `claude --version` absent)

Mirror the 2A.2 scenario: real `startOrchestratorServer` + MCP client +
real worktree, `spawn_worker({role:'researcher'})` with a trivial
no-tools task. Assert the worker emits a `result` event with **no
`parse_error`** — proving the multi-KB composed prompt is a valid
stream-json first frame the real CLI accepts (first-frame-requirement
regression guard) — then `kill_worker` → terminal status.

## Out of scope

- Composer extraction/substitution mechanics (covered by
  `tests/workers/prompt-composer.test.ts`).
- Multi-role / sibling / project-command sourcing (covered by
  `tests/integration/4a-worker-prompt.integration.test.ts`).
- Worktree-`CLAUDE.md` file injection + `[NEW TASK]` staleness guard +
  `.git/info/exclude` (deferred to Phase 4D.2).
