# Phase 9D scenario — browser plugin security envelope enforced end-to-end

**Given** a Symphony orchestrator server with the plugin host enabled
(`pluginsEnabled: true`) and a real `requires:host-browser-control` fixture
plugin installed + enabled (`chrome-devtools-skeleton`, mirroring the
`chrome-devtools-mcp` example's envelope: `requires:host-browser-control` +
`irreversible`, `toolScope: "act"`). The plugin exposes one non-functional
skeleton tool, `navigate_page`.

Setup (mirrors `tests/scenarios/7a.test.ts`):
- `SYMPHONY_PLUGINS_DIR` → `tests/fixtures/plugins`
- a temp `config.json` with `{ schemaVersion: 1, pluginsEnabled: true }`
- `SqlitePluginStore.upsert({ id: 'chrome-devtools-skeleton', enabled: true })`
- boot via `startOrchestratorServer({ initialMode: 'act', initialTier: N, plugins: { enabled: true } })`
- connect a Maestro-equivalent MCP `Client` over an in-memory transport

**When** the Maestro-equivalent client lists tools and calls
`chrome-devtools-skeleton__navigate_page` at different autonomy tiers.

**Then** (observable — audit rows + tool results, never self-reports):

1. **Discovery is tier-independent.** `listTools()` contains the namespaced
   proxy `chrome-devtools-skeleton__navigate_page` regardless of tier.
2. **Tier 2 → DENIED.** The call returns `isError: true` with a reason matching
   `/tier 3/i`, and the non-defeatable audit log has a `tool_denied` row for the
   tool. (`requires:host-browser-control` is EXACT Tier 3 — a Tier-2 session
   cannot reach it.)
3. **Tier 3 (act, present, non-automation) → ALLOWED.** The call succeeds,
   returns the `{ implemented: false }` skeleton marker, and the audit log has a
   `tool_called` row for the tool.

This proves the security envelope is **enforced by the existing
`capabilities.ts` dispatch path** purely from the manifest's `capabilityFlags` —
the plugin (and Phase 9D) ships **zero** enforcement code. The away-mode,
automation-context, and plan-mode denials are covered against the real
`CapabilityEvaluator` in `tests/plugins/9d-browser-skeletons.unit.test.ts`.
