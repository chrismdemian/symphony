# Phase 3J — Diff Preview production scenario

## What this scenario verifies

End-to-end through `runStart`:

1. Selecting a worker in the output panel and pressing `D` toggles to the
   diff view.
2. The diff view fetches via the new `workers.diff` RPC procedure.
3. The header `Diff vs <branch>@<sha>` appears in stdout.
4. Pressing `D` again toggles back to the streaming output view.

## Inputs

- Fake Maestro (no real Claude subprocess); fake RPC client whose
  `workers.list` returns one worker; `workers.diff` returns a canned
  result.
- Real keybind dispatcher and TUI tree under `runStart`.

## Acceptance

- After `D` keystroke, stdout includes `Diff vs main@`.
- After a second `D`, stdout no longer renders the diff header in the
  most recent frame.
