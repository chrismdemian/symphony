# Phase 2B.2 — Typed RPC contract + WebSocket transport (production scenario)

**Goal:** Prove that a real `symphony mcp-server` subprocess advertises its
WS-RPC bind on stderr, persists a `rpc.json` descriptor with a fresh
token, and serves the typed RPC surface end-to-end. The `IpcClient<SymphonyRouter>`
proxy returned by `createRpcClient` round-trips against the real binary.

This is the authoritative end-to-end test for 2B.2 — distinct from the
integration test which runs the orchestrator in-process. Here we spawn
`tsx src/index.ts mcp-server --in-memory --rpc-port 0` as a separate
subprocess, parse the advert from its stderr, connect a real WebSocket,
and walk the typed surface.

## Steps

1. Init a fresh git repo at `<sandbox>/repo` with a single committed
   `README.md` (so `defaultProjectPath` has a valid project to inherit).
2. Spawn `tsx src/index.ts mcp-server --in-memory --rpc-port 0
   --rpc-token-file <sandbox>/rpc.json` with `cwd=<sandbox>/repo`.
3. Capture stderr line-by-line. Wait for a line matching the
   `[symphony] {...}` advert with `event: "symphony.rpc.ready"`.
4. Parse the advert — extract `host`, `port`, `tokenFile`.
5. Read `<sandbox>/rpc.json` and confirm the descriptor contains the
   same host+port and a non-empty `token` string. Also confirm `pid`
   matches the subprocess's PID.
6. `RpcClient.connect` against `ws://<host>:<port>` with the token from
   the descriptor file. Handshake completes.
7. `client.call.projects.list()` — assert at least one project (the
   default-project synthesized by the orchestrator).
8. `client.call.projects.register({ name: 'scenario-proj', path: '/tmp/whatever' })`.
9. `client.call.projects.list()` — assert `scenario-proj` is now present.
10. `client.call.tasks.create({ projectId: 'scenario-proj', description: 'demo' })`.
11. `client.call.tasks.list({ projectId: snap.projectId })` — assert one task.
12. `client.call.mode.get()` — assert `{ mode: 'plan' }` (default).
13. Disconnect WS client. Send SIGTERM to the subprocess. Wait for
    exit. Assert the descriptor file at `<sandbox>/rpc.json` is removed
    (cleanup ran in `handle.close()`).
14. Cleanup: rm sandbox.

## Observable-only assertions

- The advert line on stderr is single-line, JSON-parseable, has
  `host: '127.0.0.1'`, integer `port`, non-empty `tokenFile`.
- Descriptor file mode is `0o600` on POSIX (skipped on Win32).
- `IpcClient<SymphonyRouter>` calls round-trip with no manual
  serialization on the test side — typed args, typed returns.
- `projects.list` reflects writes from `projects.register` immediately.
- `tasks.create` returns a snapshot whose `projectId` matches the
  persisted project's `id`.
- After SIGTERM, the descriptor file is removed within ~3 seconds.

## What this scenario proves

- `symphony mcp-server` binds the WS-RPC server alongside the MCP-stdio
  surface in one process (Phase 2B.2 single-engine invariant).
- The `[symphony] {...}` advert is parseable by external clients (TUI,
  test harnesses) without scraping `~/.symphony/rpc.json`.
- The descriptor file is created with the right shape AND removed on
  clean shutdown.
- `createSymphonyRouter` + `IpcClient<SymphonyRouter>` deliver typed
  end-to-end RPC over a real loopback WebSocket.
- `--rpc-token-file` flag honored — cleanup deletes the file the user
  pointed it at.
