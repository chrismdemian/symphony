# Phase 3H.1 production scenario

End-to-end exercise of the read-only settings popup through `runStart`,
with a fake Maestro process and a fake RPC client. Validates the full
keystroke pipeline:

- StatusBar mounts → `Ctrl+,` pushes the settings popup → settings popup
  renders all six section headers + every value field with the
  `(default)` annotation when no `~/.symphony/config.json` exists →
  `Esc` pops back to the main split.
- Slash command `/config Enter` from the chat input pushes the same
  popup → `Esc` pops back.
- A second pass with `SYMPHONY_CONFIG_FILE` pointing at an on-disk
  `config.json` containing `{ schemaVersion: 1, modelMode: "opus",
  maxConcurrentWorkers: 8 }` → `Ctrl+,` → popup renders `opus` and `8`
  with the `(from file)` annotation in violet.

Mirrors `tests/scenarios/3f1.test.ts`'s shape — same FakeMaestroProcess,
same `runStart` boot, same Ctrl-byte injection through the launcher's
stdin pipeline. Differs only in:
- `SYMPHONY_CONFIG_FILE` env override pointing at a tmp file (so the
  scenario doesn't pollute the user's real `~/.symphony/`).
- Asserts the popup body content rather than worker/question popups.

## Steps

1. Boot the launcher with a fake Maestro and a fake RPC, no workers, no
   questions, with `SYMPHONY_CONFIG_FILE` set to a non-existent tmp path.
2. Settle one polling tick.
3. Write `\x1b,` (Ctrl+,) to stdin — actually `0x2c` after Ctrl is the
   raw byte `0x1c` (`Ctrl+\\`); but Ctrl+, has no canonical low-ASCII
   mapping, so we deliver the chord through the `useInput` `key.ctrl`
   path: the Ink stream parser maps `\x1b[44;5u` (kitty CSI-u for
   Ctrl+,) onto `{key.ctrl: true, input: ','}`. In the scenario, the
   simpler path is to send the slash command form (`/config\r`) which
   takes the SAME settings popup path through `slashCommands.ts:60`.
4. Settle.
5. Assert stdout contains `Settings · Phase 3H.1 (read-only)` AND each
   of the six section labels.
6. Assert each default value is present: `mixed`, `4`, `symphony`,
   `true`, `false`, `(none)`, `300`, `1`, `0 entries`.
7. Assert the source line: `Source: (no file — using defaults)`.
8. Write `\x1b` (Esc) to dismiss the popup.
9. Settle.
10. Assert chat panel placeholder is back.
11. Write a `~/.symphony/config.json` containing customized fields
    (modelMode=opus, maxConcurrentWorkers=8).
12. Send `/config\r` again.
13. Settle.
14. Assert popup now shows `opus` and `8` with `(from file)`
    annotations.
15. Stop the launcher cleanly.

## Acceptance assertions

- After step 5: `Settings · Phase 3H.1 (read-only)` rendered; six
  section labels rendered.
- After step 7: source line shows the `(no file — using defaults)`
  variant.
- After step 10: chat panel placeholder text rendered (popup unmounted).
- After step 14: rendered popup contains `opus` (one of the file-source
  values) AND `(from file)` annotations.
- After teardown: `rpc.close` called.
