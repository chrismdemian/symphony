# Phase 4G.2 scenario — verify_ui MCP tool through the dispatch surface

## Goal

End-to-end verification that the production `makeVerifyUiTool` MCP tool:
- Refuses (structured error code) when the project has no `previewCommand`.
- Refuses when the project has no UI stack.
- On a properly-configured UI project, boots the preview (stubbed launcher),
  runs the screenshotter (stubbed; writes a fake PNG), and returns the
  paths in the MCP response's `structured_content`.
- Wires through the real `WorkerRegistry` + `ProjectRegistry` + tool
  handler shape Maestro will invoke.

Ground Truth Is Observable: assertions are on the MCP response shape
(`isError`, `structured_content.code` / `screenshot_paths`) — not on
any worker self-report. The OS boundary (real preview subprocess + real
chromium) is stubbed via the DI seam (`previewLauncher` + `screenshotter`),
following the Track-1 boundary used by `tests/scenarios/4e.test.ts` +
`tests/scenarios/4g1.test.ts`.

## Given

- A real `WorkerRegistry` + a real `ProjectRegistry`.
- A worktree (temp dir) seeded with a `package.json` declaring React so
  `detectUiStack` returns `hasUiStack: true`.
- The REAL `makeVerifyUiTool` wired with DI fakes for the launcher +
  screenshotter (the same seam unit tests use).

## When / Then

### Section 1 — no previewCommand → structured error

A project registered WITHOUT a `previewCommand`. Worker spawned against it.
Invoking `verify_ui(worker_id)`:

- `isError: true`
- `structured_content.code === 'no-preview-command'`
- Response text mentions setting `previewCommand`.

### Section 2 — no UI stack → structured error

A project registered WITH a `previewCommand` but the worktree's package.json
declares only a non-UI dep (lodash). Invoking `verify_ui`:

- `isError: true`
- `structured_content.code === 'no-ui-stack'`

### Section 3 — happy path → screenshot_paths in structured_content

A project with `previewCommand` AND a UI stack. Launcher returns a fake
URL handle; screenshotter writes a fake PNG file per viewport.

- `isError: false`
- `structured_content.ok === true`
- `structured_content.preview_url === 'http://localhost:5173/'`
- `structured_content.screenshot_paths.desktop` + `.mobile` are non-empty
  absolute paths under `<worktree>/.symphony/screenshots/<iso>/`.
- The PNG files exist on disk after the call.

## Acceptance

Pass condition: all Section 1–3 assertions hold via the real
`makeVerifyUiTool` handler. Deterministic; no real `claude -p` required
and no real Playwright required.
