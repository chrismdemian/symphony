# Phase 3B.3 scenario — Animations + status line through the launcher

## What this exercises

A full chat turn driven through `runStart`, asserting that the status line
(Equalizer + ShimmerText verb) appears in the rendered TUI buffer while a
tool is in flight, and disappears after `turn_completed`.

## Coverage limits

- `useAnimation` ticks via real timers in this scenario. The launcher's
  hook server uses `setImmediate` and microtask resolution chains; the
  `toFake` config that drives `useAnimation` correctly under unit tests
  doesn't fake those queues, so applying it here would deadlock the
  bootstrap. We therefore only assert "EQ glyph + verb present" and
  "absent after completion", NOT specific frame indices. Those are
  covered in unit tests via `vi.useFakeTimers()`.
- The kitty keyboard render option is asserted via the `runTui.test.tsx`
  unit test, not at the scenario level — we'd need a real PTY to observe
  the actual escape sequence emitted on enter.

## Why fileParallelism: false

`vitest.scenarios.config.ts` already enforces this for the whole suite.
Real-claude scenarios race on `~/.claude.json` trust file injection;
3B.3 doesn't spawn `claude` but inherits the global setting.
