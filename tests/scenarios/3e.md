# Phase 3E production scenario

End-to-end exercise of the question queue surface via `runStart`, with
a fake Maestro process and a fake RPC client. Validates the full data
path: queue starts empty → StatusBar shows `Q: 0` → RPC list flips to
one blocking question → poll tick → StatusBar shows `Q: 1` (red) →
Ctrl+Q → popup mounts → user types + Enter → `rpc.call.questions.answer`
fires → list returns empty → popup auto-pops → StatusBar `Q: 0`.

Mirrors `tests/scenarios/3d2.test.ts`'s shape; differs in the RPC method
under test (questions instead of workers) and in the keystroke driving
(Ctrl+Q to open popup, character keys + Enter to answer).

## Steps

1. Boot the launcher with a fake Maestro and a fake RPC.
2. Initial `questions.list({answered: false})` returns `[]`.
3. Wait one settle window; StatusBar shows `Q: 0`.
4. Flip the fake RPC's `questions.list` to return one blocking question
   `{ id: 'q-100', urgency: 'blocking', question: 'Postgres or
   SQLite?', projectId: 'p1', workerId: 'w-eng' }`.
5. Wait one poll tick (>1s) for `useQuestions` to refresh; StatusBar
   shows `Q: 1` (red).
6. Call `rpc.call.questions.answer({ id: 'q-100', answer: 'deploy
   us-east' })` directly — this simulates what the popup's submit path
   does end-to-end. (The keystroke chain Ctrl+Q → popup → Enter through
   Ink's stdin is covered by `tests/ui/panels/questions/
   QuestionPopup.test.tsx` + `tests/ui/keybinds.global.test.ts`. Driving
   raw bytes through a `PassThrough` into Ink's input handler from this
   test process isn't reliable; the launcher pipeline is what the
   scenario gates.)
7. Fake answer mock removes the question from the queue.
8. Wait one poll tick; StatusBar shows `Q: 0` again.
9. Stop the launcher cleanly.

## Acceptance assertions

- After step 3: rendered stdout contains `Q: 0`. `questions.list`
  called at least once.
- After step 5: rendered stdout contains `Q: 1`.
- After step 6: `rpc.call.questions.answer` invoked with
  `{ id: 'q-100', answer: 'deploy us-east' }`.
- After step 8: rendered stdout (most recent slice) contains `Q: 0`.
- `rpc.close()` called on launcher stop.
