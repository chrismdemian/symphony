# Phase 3F.3 production scenario

End-to-end exercise of the question-history popup surfaced via the
3F.1 command palette. Validates that:

- The new `questions.viewAnswered` palette command (key kind `'none'`,
  scope `'global'`) is listed in the palette and Enter fires its
  `onSelect` (which pushes the `'question-history'` popup).
- `<QuestionHistory>` mounts at scope `'question-history'`, fetches
  `rpc.call.questions.list({ answered: true })`, and renders the
  results.
- Esc closes the popup and the main split re-renders (mounted-from-
  scratch — popup still uses 3E's unmount-on-popup pattern; the
  absolute-overlay refactor was attempted and reverted; see
  `Layout.tsx:51-66` and Known Gotchas 3F.3 for the rationale).

## Steps

1. Boot the launcher with a fake Maestro and a fake RPC, one project,
   one answered question in the history.
2. Settle one polling tick.
3. Verify chat panel input placeholder rendered (proves chat mounted).
4. Write `\x10` (Ctrl+P) → palette opens.
5. Type `answered` → palette filters to the `view answered questions`
   command.
6. Press `\r` (Enter) → palette closes; QuestionHistory popup mounts.
7. Assert popup contains "Answered questions" header AND the seeded
   answer text (`us-east-1`).
8. Press `\x1b` (Esc) → popup closes; chat panel input placeholder
   re-renders in the post-Esc frame.
9. Stop launcher cleanly.

## Acceptance assertions

- After step 5: `view answered questions` row visible in palette.
- After step 7: `Answered questions` header AND seeded answer text
  (`us-east-1`) visible in post-Enter frame.
- After step 8: chat panel marker (`Tell Maestro what to do`) in
  post-Esc frame.
- `rpc.close()` called on launcher stop.

## Out of scope (deferred)

- The absolute-positioned overlay refactor that would preserve
  underlying body state (chat scroll, output stream, worker selection)
  across popup open/close cycles. Attempted in this sub-phase; Ink's
  cell-based renderer doesn't fully mask body content where popup
  cells are whitespace. Deferred with a different mechanism (state-
  lift to provider context layer) — see Known Gotchas 3F.3.
