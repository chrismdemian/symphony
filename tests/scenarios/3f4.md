# Phase 3F.4 production scenario

End-to-end exercise of the markdown-fence detector + syntax highlighter
+ diff colorizer through the launcher's worker-output panel. Validates
that:

- A worker emits `assistant_text` events containing fenced ts code +
  fenced diff blocks (alongside ordinary narrative).
- The output panel (with one worker selected) renders:
  - The ts code block tokenized: `const`/`number` keywords, the literal
    `'hello'` as string, the inline number `42`.
  - The diff block line-colored: `+` lines, `-` lines, `@@` hunk header.
  - Any surrounding narrative text in the default output color.
  - A reserved `json-render` fence still flows through the upstream
    `jsonRenderDetect` pass (Phase 3D.2) — the markdown detector does
    NOT consume it.

Mirrors `tests/scenarios/3d2.test.ts` shape — same `makeFakeRpc` with a
controllable `workers.tail` returning scripted assistant_text events.

## Steps

1. Boot the launcher with a fake Maestro and a fake RPC.
2. Settle (no workers yet).
3. Add one worker to the registry; wait for the next 1s reconcile poll
   so output panel mounts WorkerOutputView, subscribe + tail fire,
   backfill resolves.
4. Backfill returns three scripted assistant_text events:
   - Plain narrative
   - A ts code block: \`\`\`ts\nconst greeting: string = 'hello';\nconst n: number = 42;\n\`\`\`
   - A diff block: \`\`\`diff\n@@ -1,2 +1,2 @@\n-old\n+new\n\`\`\`
5. Assert rendered stdout:
   - Contains the narrative literal.
   - Contains `const`, `'hello'`, `42` (token-rendered, ANSI stripped).
   - Contains `+new` and `-old` and `@@`.
6. Stop launcher cleanly.

## Acceptance assertions

- After step 5: rendered text shows all expected strings.
- `tailMock` called once with `{ workerId, n: 200 }`.
- `rpc.close()` called on launcher stop.

Truecolor ANSI verification (violet keyword, gold string, green/red
diff lines) is covered by the visual harness — `vitest.scenarios.config.ts`
loads no `setupFiles` so chalk runs at level 0; only content matters
here.
