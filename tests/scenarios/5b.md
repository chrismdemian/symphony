# Phase 5B — Production scenario: `symphony add` / `list` / `remove` CLI round-trip

## Given

- A clean sandbox dir on disk.
- A SQLite DB path that does NOT yet exist (fresh-machine state).
- A real git repo at `<sandbox>/demo-app/` initialized via `git init` + an initial commit.
- That repo contains a `package.json` with `{ "name": "demo-app" }`.
- That repo contains a `.symphony.json` with a `project` overlay:
  ```json
  {
    "project": {
      "qualityPipeline": "simplified",
      "defaultAutonomyTier": 2,
      "previewCommand": "pnpm dev"
    }
  }
  ```

## When

A user runs three Symphony CLI commands in order, with the same `SYMPHONY_DB_FILE`:

1. `runAdd({ projectPath: '<sandbox>/demo-app' })`
2. `runList({ format: 'json' })`
3. `runRemove({ nameOrId: 'demo-app' })`

Between the second and third, raw SQL inspects `projects` to verify the overlay round-trip.

## Then

1. **After `runAdd`**:
   - `ok === true`, `project.name === 'demo-app'`, `project.path === <resolved sandbox path>/demo-app`.
   - The DB file now exists (migrations applied on fresh DB).
   - Raw `SELECT * FROM projects WHERE id = 'demo-app'` returns one row with
     `quality_pipeline = 'simplified'`, `default_autonomy_tier = 2`,
     `preview_command = 'pnpm dev'`.
2. **After `runList`**:
   - `ok === true`, `projects.length === 1`.
   - The JSON written to stdout parses to a one-element array whose `name === 'demo-app'`
     and whose `qualityPipeline === 'simplified'`.
3. **After `runRemove`**:
   - `ok === true`, `removed.name === 'demo-app'`.
   - Raw `SELECT COUNT(*) FROM projects` returns 0.
   - Subsequent `runList({ format: 'json' })` prints `[]`.

## Notes

- Real `git init` + real SQLite + real `.symphony.json` on disk. No mocks.
- Exercises the CLI runners as imported from `src/cli/{add,list,remove}.js` — same path
  `src/index.ts` calls when the user types `symphony add`/`list`/`remove`.
- The active-children + `--force` branches of `runRemove` are covered by the unit tests
  (`tests/cli/remove.unit.test.ts`) — this scenario covers the user's happy path.
