# Phase 2B.1 — SQLite state layer (production scenario)

**Goal:** Prove that a real `node dist/index.js mcp-server` subprocess can:
1. Open a SQLite DB at `SYMPHONY_DB_FILE`
2. Apply migrations from the bundled `dist/migrations/`
3. Validate the schema contract
4. Serve MCP tools backed by SQLite-persisted stores
5. Survive a full subprocess kill + restart, with data intact

This is the authoritative end-to-end test for 2B.1 — distinct from the
integration test which uses `InMemoryTransport`. Here we launch a real
child process, talk to it over stdio, kill it, launch again with the
same `SYMPHONY_DB_FILE`, and verify the state is recoverable.

## Steps

1. Build the binary (`pnpm build`). `dist/index.js` and `dist/migrations/`
   both exist.
2. `mkdtemp` a scratch dir; set `SYMPHONY_DB_FILE` inside it.
3. Spawn `node dist/index.js mcp-server` as a child with that env var.
4. Connect via stdio + `StdioClientTransport`.
5. Register a task via `create_task`. Capture its id.
6. Close the client, kill the subprocess.
7. Confirm the DB file exists on disk (non-zero bytes).
8. Spawn a SECOND subprocess with the same env var.
9. Call `list_tasks` — the originally-created task must appear with the
   exact id, description, and priority.
10. Close. Remove scratch dir.

## Observable-only assertions

- DB file exists after first run (byte count > 0).
- Second run's `list_tasks` returns the previously-created task with
  `id`, `description`, and `priority` preserved.
- Second server's `list_projects` includes the default project (seeded
  from `defaultProjectPath`).
- No process lingers — the kill is idempotent.
- Worker subprocess `exitCode === null` until we kill it (per emdash —
  signaled death on Win32 is surfaced via `signal`, not `exitCode`).

## What this scenario proves

- End-to-end the CLI subcommand wiring works (`mcp-server` opens DB by default).
- tsup correctly copies migrations to `dist/migrations/`.
- `resolveDatabasePath()` honors `SYMPHONY_DB_FILE`.
- Schema migrates to the expected shape on first open, is idempotent on second.
- Tool dispatch works through the SQLite-backed store set.
- State persists across a full process-boundary restart.
