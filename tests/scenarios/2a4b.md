# Phase 2A.4b — Production Scenario

## Goal

Exercise `audit_changes` and `finalize` end-to-end against real git state — real bare
remote, real clone, real worker worktree, real commit/push/merge — with a fake
`oneShotRunner` feeding reviewer responses. Covers the happy path (push only),
the merge path at tier 3, the audit-FAIL short-circuit, and the signal-abort
contract.

## Setup

```
tmp/
├── remote.git/              # bare remote created with git init --bare
├── main-repo/               # clone of remote.git, registered as the project
│   └── .symphony/worktrees/
│       └── wk-fake-1/       # real worktree on a feature branch
└── ...
```

- `ProjectRegistry` holds a single `main` project pointing at `main-repo`.
- `lintCommand`/`testCommand`/`buildCommand` set to `node -e "process.exit(0)"` (cheap,
  exit-0) so the finalize pipeline exercises every shell step.
- `verifyCommand` set to a short node one-liner for the happy path; override to
  `node -e "process.exit(1)"` for the FAIL path; override to a `setTimeout`-based
  script for the timeout path.
- Fake `oneShotRunner` returns a canned JSON envelope with `verdict: PASS` by
  default; swapped at runtime for FAIL + malformed cases.

## Assertions

1. **audit_changes (PASS)** — returns `verdict: PASS` with the canned findings,
   `isError: false`.
2. **finalize no-merge (happy)** — returns `ok: true`, `commitSha` populated,
   `mergeSha` absent. Remote tracking branch `origin/feature/<id>` exists in
   `main-repo` after the push.
3. **finalize with merge_to=main (tier 3, distinct worktree)** — returns
   `ok: true`, `commitSha` + `mergeSha` populated. `git log origin/main` in
   `main-repo` includes a merge commit. Feature branch deleted from remote.
4. **audit FAIL aborts at step 1** — finalize returns `ok: false`, `failedAt: 'audit'`,
   NO commit made (git status in worktree unchanged).
5. **Signal abort mid-run** — with a long-running `verifyCommand`, aborting the
   `AbortController` produces an `aborted` outcome; no commit.
