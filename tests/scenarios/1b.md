# Phase 1B — Production Scenario

WorkerManager drives a real `claude -p` subprocess through a single-turn conversation, captures the session ID, and resumes that session in a second worker.

## Given

- A `claude` binary is resolvable via PATH (`claude --version` returns 0). If it is not, this scenario **skips with a warning** so CI without claude still passes.
- A fresh temporary working directory (the "worktree") that the user has NOT trusted in `~/.claude.json` yet.
- The Phase 1B `WorkerManager` public API (`spawn`, `worker.events`, `worker.waitForExit`, `worker.sessionId`).

## When

1. Caller pre-injects trust for the temp directory via `ensureClaudeTrust` (indirectly, through `WorkerManager.spawn`).
2. Caller spawns a first worker with a minimal prompt: *"Reply with exactly the word 'ok' and nothing else."*
3. Caller iterates the worker's event stream to exhaustion and awaits `waitForExit`.
4. Caller captures the `sessionId` reported in the final `result` event.
5. Caller spawns a SECOND worker with the SAME cwd and `sessionId` set — triggering the `--resume <uuid>` path and Phase 1B's `validateResumeSession` check (which expects `~/.claude/projects/<encoded>/<uuid>.jsonl` to exist by now, because the first worker just created it).
6. Caller iterates the second worker's events and awaits exit.

## Then

Observable state:

- First worker exits with `status === 'completed'` and `exitCode === 0`.
- First worker's event sequence contains at least `system_init` and `result`.
- First worker's `result.sessionId` is a UUID-shaped string.
- First worker's `result.isError === false`.
- No `parse_error` events are emitted for either worker.
- Second worker's `system_init.sessionId` equals the captured ID from the first worker (proving `--resume` was honored and the session was found on disk).
- Second worker exits with `status === 'completed'`.
- The jsonl session file at `~/.claude/projects/<encoded-cwd>/<sessionId>.jsonl` exists at the end (Claude wrote it).

## Ground Truth Is Observable

Every assertion is derived from Symphony-observable outputs: `Worker.status`, `WorkerExitInfo.sessionId`, emitted `StreamEvent`s, filesystem artifacts Claude itself wrote. No parser internals, no mocked subprocess, no assumptions about the model's exact text output beyond the `result.isError === false` contract.

## Gates

- **Skip-gracefully**: If `claude --version` fails (non-zero exit or missing binary), the test is marked `skip` with the reason logged. `pnpm verify` still passes.
- **Timeout budget**: 60 seconds per worker spawn. Longer than a typical hello-world turn but generous for a cold Claude Code startup and any OAuth re-auth prompts.
- **Network dependency**: Real claude spawns talk to Anthropic's API. Running this scenario locally consumes a small amount of Max-subscription quota — acceptable for a single acceptance check.
