# Phase 1A — Production Scenario

Stream parser survives a complete `claude -p` worker session end-to-end.

## Given

- `tests/helpers/fake-claude.mjs` stands in for `claude -p --output-format stream-json --verbose` by replaying a fixture file over stdout.
- Fixtures under `tests/fixtures/stream-json/` represent full sessions captured in the on-the-wire shape.

## When

1. A caller (Symphony's future WorkerManager) spawns `fake-claude structured-completion` via `child_process.spawn`.
2. The caller pipes `child.stdout` through `parseStream()` and consumes events via `for await`.
3. The subprocess emits lines in real time; caller does NOT wait for process exit before consuming.

## Then

Observable state:

- Exit code is `0` and stderr is empty (no subprocess crash).
- Events arrive in order: `system_init` → `assistant_text` → `structured_completion` → `result`.
- `system_init.sessionId` is a UUID-shaped string.
- `system_init.tools` includes at least one tool name.
- `assistant_text.text` is non-empty.
- `structured_completion.report.audit` is `"PASS"` or `"FAIL"`.
- `structured_completion.report.did` is a non-empty string array.
- `structured_completion.report.cite` is present (may be empty).
- `result.sessionId` matches `system_init.sessionId`.
- `result.costUsd` is a finite number ≥ 0.
- `result.usageByModel` contains a per-model entry with positive `inputTokens`.
- No `parse_error` events are emitted for this fixture.

## Ground Truth Is Observable

Every assertion above is derived from the event stream produced by the subprocess — never from the worker's self-reported claims. The test binds exit code, stderr, and emitted event sequence; the parser's internals are irrelevant to the contract.
