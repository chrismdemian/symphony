# 3B.1 Production Scenario — Chat Panel MVP (assistant-streaming path)

**Sub-Phase Acceptance Gate 3.** Real `runStart` engagement with a
TTY-flagged stdout, fake Maestro process, and an end-to-end assertion
that streamed assistant chunks reach the rendered frame buffer through
the new chat panel + provider plumbing.

## Coverage scope

This scenario covers the **assistant-streaming → render** half of the
chat round-trip. The user-input → `sendUserMessage` half is covered at
the unit level (`tests/ui/panels/chat/InputBar.test.tsx` and
`ChatPanel.test.tsx`) using `ink-testing-library`'s purpose-built stdin
mock. A vanilla `PassThrough` doesn't cleanly decode keystrokes through
Ink's input parser end-to-end at the `runStart` layer (Ink's
`ink-testing-library` adds a custom `Stdin` class that fires both
`'readable'` and `'data'` events for each `write`, which we'd have to
reproduce). Tracked as a 3B.2 follow-up — when paste / scroll / slash
land, port the keystroke harness to the scenario level.

## Given

- `runStart` invoked with TTY-flagged stdin/stdout PassThroughs
  (mirrors the 3A scenario harness).
- `FakeMaestroProcess` substituted via `maestroFactory`. The fake
  records `sendUserMessage` calls and exposes an `events()` async
  iterator over its emitter.
- RPC fully stubbed (no real WebSocket).
- Project list pre-seeded with one demo project.

## When

1. Launcher boots; TUI mounts; chat panel renders the empty state.
2. The Plan-agent S3 invariant is asserted: ONE iterator opened by the
   provider (`fake.eventsIterCount === 1`).
3. The fake emits a streaming reply: `turn_started` → three
   `assistant_text` chunks (`"Hi", " there", "!"`) → `turn_completed`.
   Each emit is followed by a microtask flush so the iterator's
   parked `next()` resolves before the next event lands.
4. After ~300 ms settle (Ink's render throttle is ~32 ms), the
   accumulated stdout buffer is grepped for the printable text.

## Then

- The assistant reply `"Hi there!"` appears in the rendered frame
  (text-block coalescing into a single bubble; no chunk fragmentation).
- `fake.eventsIterCount === 1` (single-iterator invariant).
- `handle.stop()` tears down cleanly: hook uninstalled, fake killed
  once, RPC closed.

## Failure modes guarded by adjacent tests

- Multi-iterator race (Plan-agent S3) — asserted directly here.
- O(N²) chunk-join cost (Plan-agent A2) — covered by the reducer's
  unit test (`chatHistoryReducer.test.ts` "coalesces consecutive
  assistant_text").
- `MaestroTurnInFlightError` surfacing — covered by the ChatPanel unit
  test ("surfaces ... as an inline error message").
- User-bubble rendering — covered by `ChatPanel.test.tsx` ("sends a
  typed message and renders the user bubble") via
  `ink-testing-library`.
