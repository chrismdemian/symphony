# Phase 4D scenario ‚Äî fragment composition, worktree injection, skills

## Goal

End-to-end verification that the five 4D sub-phases work together
through real paths (real `spawn_worker` MCP tool ‚Üí real
`createWorkerLifecycle` ‚Üí real `WorktreeManager` ‚Üí real git; real fs for
skills + JSONC). Ground truth is observed bytes on disk and in
`cfg.prompt`, never a self-report.

## Given

- A registered project at a temp git repo (real `git init`, one commit).
- The real frozen v1 artifacts + generated fragments at
  `research/prompts/` + `research/prompts/fragments/` (no override).
- An isolated skills store via `SYMPHONY_SKILLS_DIR` /
  `SYMPHONY_CLAUDE_COMMANDS_DIR` env overrides ‚Üí temp dirs.

## When / Then

### Section 1 ‚Äî 4D.1 + 4D.2 (fragment compose + worktree injection)

Drive the REAL `spawn_worker` tool handler into a REAL worktree. The
`claude` subprocess is stubbed to capture `cfg`.

- `<worktree>/CLAUDE.md` exists, starts with the Symphony worker
  sentinel, and contains `## Your Role: Implementer` +
  `### Reporting Format ‚Äî MANDATORY` (the fragment-composed manual,
  Multica-injected).
- `cfg.prompt` (worker stdin) is **task-only** ‚Äî `# Your Task\n\n<task>`
  with no role text and no sentinel.
- `git status --porcelain` in the worktree does NOT list `CLAUDE.md`
  (the `.git/info/exclude` written to `--git-common-dir` keeps the
  injected file out of commits ‚Äî the 4D.2 latent-1C fix).

### Section 2 ‚Äî 4D.3 + 4D.4 (skills store + bundled installer)

- `installBundledSkills()` installs `dev-browser` + `json-render` into
  the isolated central store, both linked; a second run skips both
  (idempotent).
- A custom skill install ‚Üí list (linked) ‚Üí uninstall roundtrip leaves
  the store empty; uninstall NEVER removes a real (non-symlink) user
  directory at the agent path.

### Section 3 ‚Äî 4D.5 (format-preserving JSONC)

- `editJsoncFile` on a `settings.local.json`-shaped file with a user
  comment edits a nested path while preserving the comment + sibling
  keys; `stripOwnEntriesByMarker` drops Symphony-owned hook entries and
  keeps user ones.

## Acceptance

All three sections pass deterministically with no real `claude -p`
required (the new 4D behavior is fs/composition, fully observable).
