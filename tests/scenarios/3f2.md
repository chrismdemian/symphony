# Phase 3F.2 production scenario

End-to-end exercise of the leader-key chord plumbing through `runStart`.
Validates:

- `Ctrl+X` arms the leader window; the bottom keybind bar shows the
  `Ctrl+X _` waiting hint.
- `Ctrl+X m` fires the model-mode toast.
- The toast text is rendered above the keybind bar.
- The toast auto-dismisses after the default ttl.
- Pressing `Ctrl+X` then waiting past the timeout clears the leader
  state — a subsequent `m` does NOT fire the model-mode toast.

Mirrors `tests/scenarios/3f1.test.ts` shape.

## Steps

1. Boot the launcher with a fake Maestro and a fake RPC.
2. Settle one polling tick.
3. Write `\x18` (Ctrl+X) → assert keybind bar shows `Ctrl+X _`.
4. Write `m` → assert toast text appears.
5. Wait > toast ttl → assert toast dismissed.
6. Write `\x18` again, then wait > leader timeout, then write `m` —
   assert NO toast renders (leader cleared, second keystroke ignored).
7. Stop launcher cleanly.

## Acceptance assertions

- After step 3: rendered stdout contains `Ctrl+X _` (bar shows armed
  hint, regular bar contents replaced).
- After step 4: rendered stdout contains "Model mode switch".
- After step 5: subsequent frame does NOT contain the toast text.
- After step 6: NO new "Model mode switch" toast rendered.
- `rpc.close()` called on launcher stop.
